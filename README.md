# Flastify Rest API

Учебный проект, состоящий из клиентской части (Frontend) и серверной части (Backend).

## Структура проекта

* `/frontend` — Клиентское приложение на React + Vite.
* `/server` — Серверное приложение на Fastify + Drizzle ORM + SQLite.

## Технологии

**Backend:** Fastify 5, Drizzle ORM, better-sqlite3, Zod, Pino
**Frontend:** React 18, React Router, Vite


## Быстрый старт

Для работы проекта необходимо установить зависимости в обеих директориях.

### 1. Настройка сервера

Перейдите в директорию сервера:
```bash
cd server
```

Установите зависимости:
```bash
npm install
```

Создайте миграции и примените их (если база данных еще не создана):
```bash
npx drizzle-kit generate
```

Запустите сервер в режиме разработки:
```bash
npm run dev
```
Сервер будет доступен по адресу: [http://localhost:3000](http://localhost:3000)

### 2. Настройка фронтенда

Откройте новое окно терминала и перейдите в директорию фронтенда:
```bash
cd frontend
```

Установите зависимости:
```bash
npm install
```

Запустите клиентское приложение:
```bash
npm run dev
```
Клиент будет доступен по адресу, указанному в терминале (обычно [http://localhost:5173](http://localhost:5173)).

## Доступные команды

### Сервер (`/server`)

* `npm run dev` — Запуск сервера с автоматической перезагрузкой.
* `npm start` — Запуск в продакшн-режиме.
* `npm run test` — Запуск тестов.
* `npx drizzle-kit generate` — Генерация миграций базы данных.

### Фронтенд (`/frontend`)

* `npm run dev` — Запуск Vite в режиме разработки.
* `npm run build` — Сборка проекта для продакшна.
* `npm run preview` — Предпросмотр собранного проекта.

## База данных (Drizzle)

Все команды выполняются из папки `server`.

```bash
# Сгенерировать миграцию по изменениям в db/schema.js
npx drizzle-kit generate

# Применить миграции
npx drizzle-kit migrate
```

> Если миграция сгенерировалась некорректно, её можно удалить и сгенерировать заново.

## Модель данных

- **users** — пользователи (`fullName`, `email`)
- **courses** — курсы, привязаны к создателю (`creatorId → users.id`)
- **course_lessons** — уроки, привязаны к курсу (`courseId → courses.id`)
